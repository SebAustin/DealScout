DealPulse Scout

LangGraph multi-agent car deal finder powered by Bright Data + Claude

Web Data Unlocked Hackathon · Pipeline v2

The Problem

  • Car listings are scattered across CarGurus, AutoTrader, Craigslist, and dealer sites
  • No single place answers: "What's the best deal near me for this car under my budget?"
  • Manual comparison is slow — price, mileage, market value, and location all matter
  • Buyers need ranked, explained recommendations — not just a list of links

Our Solution

Type a natural-language query. Get the top 3 ranked deals with scores and reasons.

BMW X5 Austin under 75k
Lamborghini Huracán Miami under 300k
Porsche Panamera Austin under 120k

Five LangGraph agents coordinate scraping, normalization, scoring, and ranking — streamed live to the UI.

Architecture

Demo mode synthesizes listings; live mode scrapes via Bright Data and scores with Claude.

Bright Data Integration

SERP (search_engine)

Market price research — comparable listings and price trends for scoring context.

Scraping Browser

JS-heavy sites: CarGurus, AutoTrader — full rendered page content.

Web Unlocker

Dealer sites and Craigslist — bypass blocks, return markdown.

MCP Protocol

Agents call tools via Bright Data MCP SSE endpoint — no custom scraper infra.

Claude AI Layer

  • Coordinator — interprets fuzzy queries ("75k", "near Austin", multi-word makes)
  • Normalizer — extracts structured fields from raw HTML/markdown scrapes
  • Scorer — compares listing price vs SERP market data; writes one-line reason

Heuristic fallbacks in demo/offline mode — pipeline never hard-crashes on stage.

Tech Stack

LangGraph FastAPI SSE Streamlit Bright Data MCP Claude SQLite Python 3.12

Why this stack?

  • LangGraph — explicit multi-agent orchestration with observable node events
  • SSE — judges see agents work in real time, not a black box
  • Bright Data MCP — production-grade scraping without building proxy infra
  • No Docker — one-command setup for hackathon judges

Limitations & Future Work

  • CarGurus entity IDs cover ~20 makes/models — others fall back to make-only search
  • Demo listings are samples; links open filtered search, not exact VINs
  • Live mode depends on site structure changes and rate limits

Next steps

  • Expand entity ID map + auto-discovery from CarGurus price-trends URLs
  • VIN-level matching and price history charts
  • Email alerts when a scored deal appears in live inventory
